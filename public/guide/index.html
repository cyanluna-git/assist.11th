코드 암기보다 프로젝트 이해 답변 받기보다 결과물 만들기 질문하는 사용자에서 AI에게 일을 맡기는 사용자로 이동

이 과정의 핵심

  • 비설치형으로 작은 성공 경험 만들기
  • IDE에서 여러 파일을 다루는 감각 익히기
  • WSL + CLI로 실제 개발 흐름 진입하기
  • GitHub로 프로젝트를 축적하고 공유하기

공개 가이드

  • 이 사이트는 로그인 없이 바로 열 수 있는 공개 가이드
  • Vercel 가입이나 로그인은 새 탭에서 진행하고, 설명은 현재 가이드 탭에서 계속 확인
  • 계정 가입 가이드와 3주차 배포 교안도 로그인 없이 바로 열람 가능

핵심 메시지

답변 받기보다 일 시키기

검색창처럼 묻기에서 멈추지 않기 화면 로직 데이터 버전 구분하기 구분을 바탕으로 AI에게 일 나누기

이 수업에서 하게 될 일

도구 이름보다 중요한 것 프로젝트를 이어가는 방식

어떤 AI 도구든 사용 가능 핵심은 구조 이해 화면 로직 데이터 버전 틀 안에서 더 정확하게 지시

좋은 답변보다 프로젝트 끝까지 끌고 가기

여러분이 공통으로 준비할 것

  • 공통 기반은 GitHub, Node.js, Python, WSL
  • 중간 산출물은 우선 GitHub 저장소 링크
  • 배포는 후반에 Vercel로 확장

다시 열기 · 이어서 만들기 · 기록 남기기

자유롭게 선택해도 되는 것

  • IDE는 VS Code, Cursor, Antigravity 등 자유 선택
  • AI 도구는 OpenAI, Google AI, Claude, Copilot 계열 자유 선택
  • 시연 도구와 다르더라도, 본인이 쓰는 도구로 따라와도 됨
