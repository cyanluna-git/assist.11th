3 응답 처리

4 DB 확장

다음 단계로 저장 기능 확장

0-10분

프론트엔드 백엔드 DB 다시 구분

  • 입력 폼은 프론트엔드
  • 저장 요청을 받는 코드는 백엔드
  • 기록을 오래 남기면 DB 필요

10-20분

저장 기능 UI 추가

현재 대시보드에 아래 기능을 추가해줘
1. 선택 이유 입력 칸
2. 메모 입력 칸
3. 저장 버튼
기존 화면 구조는 유지해줘

20-35분

백엔드 route 만들기

Next.js app router 기준으로
저장 요청을 받는 API route를 만들어줘
우선은 메모리 안에만 저장해도 괜찮아
초보자가 읽기 쉬운 코드로 작성해줘

35-45분

프론트에서 백엔드 호출

저장 버튼을 누르면
방금 만든 API route로 POST 요청 보내게 해줘
성공 실패 메시지도 화면에 보여줘

45-52분

왜 데이터베이스가 필요한가

  • 새로고침해도 기록 남기기
  • 여러 사용자가 함께 쓰기
  • 검색과 정렬 가능하게 만들기

52-60분

최종 정리와 다음 단계

  • 저장 기능이 붙은 화면 확인
  • GitHub에 마지막 push
  • 다음 확장 후보 정리
