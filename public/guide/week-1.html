더 나은 결과물로 반복

왜 Claude Desktop인가

  • 모든 수강생이 같은 출발점에서 시작 가능
  • 질문보다 작업지시 연습에 집중하기 좋음
  • 화면 구조와 기능 요구사항을 언어로 정리하기 좋음
  • 다음 단계에서 로컬 파일과 Claude Code로 옮기기 쉬움

보조 선택지

  • Claude 웹도 가능하지만 수업 공통 기준은 Claude Desktop
  • Replit이나 Bolt는 수업 밖 추가 실험용으로 사용 가능
  • 1주차 실습 공통 흐름은 Claude Desktop 기준으로 진행

Claude Desktop 다운로드

0-10분

질문형 사용과 작업지시형 사용 구분

  • 질문만 던지는 사용 패턴 확인
  • 원하는 결과물을 먼저 정하는 방식 소개
  • 화면 입력 결과 세 덩어리로 나누기

10-20분

프롬프트 1 첫 화면 만들기

출장 / 미팅 의사결정 대시보드를 만들어줘
한 페이지 웹앱으로 만들기
도시 날짜 국가 입력 받기
날씨 공휴일 환율 카드 3개 보여주기
처음에는 실제 API 대신 목업 데이터 사용
파일 수는 최소로 유지
초보자가 읽기 쉬운 구조로 만들기

Claude Desktop에 그대로 붙여넣기

20-35분

프롬프트 2 UI 정리

좋아
이제 아래 4가지를 고쳐줘
1. 입력 폼을 더 단순하게
2. 결과 카드 3개를 한눈에 비교 가능하게
3. 빈 값일 때 안내 문구 추가
4. 모바일에서도 읽기 쉽게

한 번에 너무 많이 바꾸지 않기

35-45분

프롬프트 3 목업 데이터 정리

좋아
이제 목업 데이터를 더 현실적으로 바꿔줘
도시마다 날씨 공휴일 환율 예시값이 다르게 보이게 해줘
로딩 문구와 오류 문구도 함께 넣어줘

45-55분

첫 산출물 직접 읽기

  • 입력 폼이 어떤 값을 받는지 확인
  • 결과 카드 3개 구성이 명확한지 확인
  • 로딩 오류 빈 상태가 설명되어 있는지 확인
