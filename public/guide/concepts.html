필수 개념

AI와 일하려면, 먼저 문제의 종류를 구분할 수 있어야 한다.

초보자에게 필요한 것은 깊은 이론 수업이 아니다. 지금 문제가 화면 문제인지, 로직 문제인지, 데이터 문제인지, 버전 문제인지를 구분하는 최소 개념이 필요하다.

Frontend

프론트엔드

사용자가 직접 보는 화면이다. 버튼, 입력창, 카드, 차트, 색상, 레이아웃이 여기에 속한다.

AI에게 이렇게 말한다: “화면만 바꿔줘”, “카드 레이아웃을 더 명확하게 해줘”

Backend

백엔드

화면 뒤에서 데이터를 처리하는 부분이다. API 호출, 계산, 권한 처리, 비즈니스 규칙이 여기에 들어간다.

AI에게 이렇게 말한다: “이 계산 로직을 함수로 분리해줘”, “API 호출 에러를 처리해줘”

Database

데이터베이스

프로젝트의 기억 창고다. 사용자 정보, 일정, 메모, 주문, 기록처럼 계속 남아야 하는 데이터가 저장된다.

AI에게 이렇게 말한다: “지금은 DB 없이 가고, 나중에 저장 기능을 붙이자”

Git

Git

파일 변경 이력을 관리하는 도구다. 무엇이 바뀌었는지 비교하고, 실수했을 때 이전 상태로 돌아갈 수 있다.

AI에게 이렇게 말한다: “이번 변경만 커밋하기 쉽게 정리해줘”

GitHub

GitHub

Git 기록과 프로젝트를 온라인에 쌓는 공간이다. 수업에서는 제출, 공유, 피드백, 코드 열람의 중심이 된다.

AI에게 이렇게 말한다: “GitHub에 올릴 수 있게 README까지 정리해줘”

반드시 기억할 문장

바이브코딩은 코드를 전부 외우는 일이 아니라, 문제의 층을 구분하는 일에 가깝다.

처음부터 알 필요 없는 것

  • 프레임워크 내부 동작 전체
  • 복잡한 서버 인프라
  • DB 최적화 심화 이론

처음부터 알아야 하는 것

  • 지금 바꾸는 대상이 화면인지 로직인지
  • 데이터 저장이 필요한지 아닌지
  • 변경 이력을 왜 남겨야 하는지
