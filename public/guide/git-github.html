핵심 문장

파일 이름으로 버전 관리 = 결국 지옥.
Git = 이름 유지 + 변경 이력 따로 누적

핵심 용어 6개

이것만 알면 오늘 실습 가능

용어 1

Repository 레포지토리

프로젝트 전체의 타임캡슐 보관함.
파일 저장 폴더가 아닌, 변경 이력까지 쌓이는 공간

# 현재 폴더를 레포로 만들기
git init

용어 2

Commit 커밋

"지금 이 상태를 저장"하는 스냅샷.
Ctrl+S = 덮어쓰기. 커밋 = 이전 상태 유지하며 새 상태 누적

# 파일을 커밋할 준비 목록에 추가
git add .

# 스냅샷 저장 (메시지 필수)
git commit -m "첫 번째 버전"

용어 3

원격 레포 Remote / Origin

GitHub의 온라인 보관함.
로컬 레포와 GitHub 레포를 잇는 이름 = origin

# GitHub 레포와 연결
git remote add origin https://github.com/assist-11th/내저장소.git

용어 4

Push 푸시

내 컴퓨터 → GitHub 업로드.
커밋 내용을 원격 레포에 반영

git push origin main

용어 5

Pull 풀

GitHub → 내 컴퓨터 내려받기.
다른 기기·협업자가 올린 내용 가져오기

git pull origin main

용어 6

Merge 머지

두 버전을 하나로 합치기.
지금은 개념만 — "다른 브랜치 작업을 메인에 합침"

# 오늘은 개념만 — 실습은 다음 강의
git merge 브랜치이름

자주 헷갈리는 것

Git과 GitHub는 다르다

Git 내 컴퓨터 안

변경 이력 관리 도구
터미널 명령어로 실행

→
GitHub 온라인 서버

기록 저장·공유 공간
브라우저로 접근하는 웹사이트

강의용 한 줄

Git = 기록 도구. GitHub = 기록 보관 장소

실습 흐름

Claude Code에게 지시하면서 하기

터미널 명령어 암기 불필요. Claude Code에게 지시 → Git 명령어 대신 실행

매번 승인 건너뛰려면 아래로 실행:

claude --dangerously-skip-permissions

자동 승인 모드 · 묻지 않고 바로 실행

0–10분 · Step 1

레포 만들고 첫 커밋

Claude Code에 붙여넣기

내 HTML 프로젝트 폴더를 git 레포로 초기화하고,
현재 파일들을 첫 커밋으로 저장해줘.
커밋 메시지는 "첫 번째 버전"으로.
  • 완료 후 git log로 커밋 기록 확인
  • .git 폴더가 생성됐는지 확인

핵심 체험 · 이력 누적 시작

10–25분 · Step 2

코드를 망치고 — 되돌리기

Claude Code에 지시

index.html의 배경색을 형광 초록으로 바꿔줘.

브라우저 확인 → 마음에 안 들면

방금 변경사항을 되돌려줘. git으로.
  • 되돌아간 것을 브라우저에서 확인
  • Claude Code가 어떤 git 명령어를 썼는지 확인

핵심 체험 · 망쳤어도 돌아올 수 있다는 안전감

25–45분 · Step 3

GitHub에 Push

assist-11th 조직 안에 빈 레포 생성 → Claude Code에 입력

GitHub에 'my-first-project' 이름으로 원격 레포를 연결하고
지금까지 커밋한 것들을 push해줘.

GitHub 레포 주소: https://github.com/assist-11th/my-first-project.git
  • GitHub에서 파일이 올라간 것 확인
  • 커밋 히스토리가 GitHub에도 보이는지 확인
  • 화면 상단 "X commits" 클릭해서 이력 탐색해보기

핵심 체험 · 프로젝트 온라인 누적

45–55분 · Step 4

작은 수정 + 두 번째 커밋

Claude Code에 입력

제목 텍스트를 내 이름으로 바꾸고
커밋 메시지 "제목 수정"으로 커밋한 뒤 push까지 해줘.
  • GitHub에서 커밋이 2개가 된 것 확인
  • 각 커밋 클릭해서 어떤 줄이 바뀌었는지 diff 확인
