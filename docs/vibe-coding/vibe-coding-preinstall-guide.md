# 바이브코딩 강의 사전 설치 안내

## 문서 목적

이 문서는 `바이브코딩 입문 강의`에 참여하기 전에 준비해야 할 항목을 정리한 사전 안내 문서다.

이번 강의는 설치 자체를 배우는 시간이 아니라,  
`AI와 함께 프로젝트를 만들고 수정하는 경험`에 집중한다.  
따라서 아래 항목은 가능하면 강의 전에 미리 준비해오는 것을 권장한다.

---

## 이번 강의에서 만들 것

첫 프로젝트는 `출장 / 미팅 의사결정 대시보드`다.

이 프로젝트에서는 아래와 같은 공개 API를 활용할 예정이다.

- 날씨 API
- 공휴일 API
- 환율 API

중요한 점:

- 이번 단계에서는 `OpenAI API Key` 같은 별도 유료 API 키가 필요하지 않다.
- `API 키 없이 바로 호출 가능한 공개 API`를 사용할 예정이다.

즉, 이번 사전 준비의 핵심은 `개발 환경`과 `작업 도구`를 준비하는 것이다.

---

## 꼭 준비해야 하는 것

### 1. GitHub 계정

왜 필요한가:

- 프로젝트를 저장하고 제출하기 위해 필요하다.
- 이번 과정의 중간 산출물은 우선 `GitHub 저장소 링크`다.

준비할 것:

- GitHub 계정 생성
- 로그인 가능 상태 확인

---

### 2. Git

왜 필요한가:

- 프로젝트 변경 이력을 관리하고 GitHub와 연결하기 위해 필요하다.

설치 후 확인:

```bash
git --version
```

---

### 3. Node.js LTS

왜 필요한가:

- 웹 프로젝트를 실행하고 패키지를 설치하기 위해 필요하다.

설치 후 확인:

```bash
node -v
npm -v
```

권장:

- `LTS 버전` 설치

---

### 4. Python 3

왜 필요한가:

- 자동화, 스크립트, 이후 확장 실습을 위해 필요하다.

설치 후 확인:

```bash
python --version
```

또는

```bash
python3 --version
```

---

### 5. WSL

대상:

- `Windows 사용자`

왜 필요한가:

- 앞으로 CLI 기반 작업을 안정적으로 하기 위해 필요하다.
- Windows 단독 환경보다 Linux 기준 명령어와 개발 흐름을 따라가기 쉽다.

권장:

- WSL 설치 후 Ubuntu 같은 배포판까지 초기 설정 완료

설치 후 확인:

```bash
wsl -l -v
```

그리고 WSL 안에서 아래를 다시 확인:

```bash
git --version
node -v
npm -v
python3 --version
```

중요:

- 가능하면 프로젝트는 `WSL 홈 디렉토리` 안에서 작업하는 습관을 들이는 것을 권장한다.

---

### 6. 에디터 또는 IDE 1개

왜 필요한가:

- 파일을 열고, 수정하고, 프로젝트를 실행하기 위해 필요하다.

가능한 예시:

- VS Code
- Cursor
- Antigravity
- 기타 본인이 익숙한 개발 도구

중요:

- 꼭 하나의 브랜드로 통일할 필요는 없다.
- 중요한 것은 아래 3가지가 가능한가다.

1. 파일 편집
2. 프로젝트 실행
3. GitHub 연결 또는 Git 사용

---

### 7. AI 작업 도구 1개 이상 로그인 상태

왜 필요한가:

- 강의의 핵심은 AI에게 질문하는 것이 아니라, 프로젝트 작업을 맡기는 경험이기 때문이다.

가능한 예시:

- OpenAI 계열 도구
- Google AI 계열 도구
- Claude 계열 도구
- GitHub Copilot

중요:

- 어떤 서비스를 쓰는지는 개인 선택이다.
- 강의에서는 대표 예시를 보여주지만, 수강생이 이미 결제해서 사용 중인 도구가 있다면 그것을 써도 된다.

---

## 권장 준비 순서

1. GitHub 계정 준비
2. Git 설치
3. Node.js LTS 설치
4. Python 3 설치
5. Windows 사용자는 WSL 설치
6. 에디터/IDE 설치
7. AI 도구 로그인 확인

---

## 수업 전 최종 확인 체크리스트

아래 항목을 모두 체크할 수 있으면 준비가 끝난 것이다.

- GitHub 계정으로 로그인할 수 있다.
- `git --version` 이 실행된다.
- `node -v` 와 `npm -v` 가 실행된다.
- `python --version` 또는 `python3 --version` 이 실행된다.
- Windows라면 `wsl -l -v` 가 실행된다.
- WSL 안에서 `git`, `node`, `npm`, `python3` 확인이 된다.
- 에디터 또는 IDE를 하나 열 수 있다.
- AI 작업 도구에 로그인할 수 있다.

---

## 수업에서 바로 하지 않을 것

이번 단계에서는 아래를 강제하지 않는다.

- OpenAI API Key 발급
- Google AI Studio API Key 발급
- Claude API Key 발급
- 유료 클라우드 배포 설정
- Vercel 배포

이런 항목은 뒤 단계에서 다룰 수 있다.

이번에는 `API 키 없이 되는 공개 API`로 충분히 실습 가능하다.

---

## 수업 당일 목표

수업이 시작되면 아래에 바로 들어갈 수 있어야 한다.

1. 공개 API를 호출하는 작은 앱 만들기
2. AI에게 기능 수정 요청하기
3. 프로젝트를 GitHub 저장소로 정리하기

즉, 수업 당일에는 `설치`보다 `실행과 수정`에 시간을 쓰는 것이 목표다.

---

## 강의 운영 메모

강의자는 아래 원칙으로 안내하는 것이 좋다.

- 공통 기반은 `GitHub`, `Node.js`, `Python`, `WSL`
- IDE와 AI 도구는 하나로 강제하지 않음
- 대표 시연 도구는 정하되, 수강생 도구 선택은 열어둠
- 이번 실습은 `공개 API` 기반이라 별도 API 키를 미리 받지 않음
